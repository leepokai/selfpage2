GraphRag代碼實踐與走過的坑

環境配置

  1. 使用虛擬環境
conda create --name myenv python=3.10 # 10-12皆可
  1. 安裝graphrag
pip install graphrag 
#需要的庫基本都包含在graphrag中了 不過如果之後要需要使用ollama的可能還要額外pip install ollama
  1. 新增資料夾
mkdir -p ./ragtest/input
  1. 初始化graphrag
python -m graphrag.index --init --root ./ragtest
  1. 設置好setting.yaml後就可以輸入以下來建構graphrag圖(如果沒設置好setting.yaml先往下看)
python -m graphrag.index --root ./ragtest

設置setting.yaml

!!!總而言之奉勸各位一句話能用openai的api就使用，不然使用本地配置的話會可能有很多的問題!!!

使用open ai api key

在.env文件中替換成自記幾的api key即可，接下來去setting.yaml中替換語言模型和embedding模型，這裡建議語言模型填入gpt4o-mini會便宜許多

<插圖>

本地使用ollama

  1. 先去ollama安裝gemma2(其他大語言模型也可)
ollama pull gemma2
  1. 再去安裝一個embedding模型
ollama pull nomic-embed-text
  1. 到setting.yaml文件中修改文件
# 第一個要改的程式碼部分
llm:
  api_key: ollama
  type: openai_chat # or azure_openai_chat
  model: gemma2
  model_supports_json: true # recommended if this is available for your model.
  max_tokens: 2048
  # request_timeout: 180.0
  api_base: http://localhost:11434/v1 
# 第二個要改的程式碼部分
embeddings:
  ## parallelization: override the global parallelization settings for embeddings
  async_mode: threaded # or asyncio
  llm:
    api_key: ollama
    type: openai_embedding # or azure_openai_embedding
    model: nomic-embed-text
    api_base: http://localhost:11434/api # 注意是 api
  1. 到graphrag\llm\openai\openai_embeddings_llm.py檔案修改成以下程式
from typing_extensions import Unpack
 
from graphrag.llm.base import BaseLLM
from graphrag.llm.types import (
    EmbeddingInput,
    EmbeddingOutput,
    LLMInput,
)
 
from .openai_configuration import OpenAIConfiguration
from .types import OpenAIClientTypes
import ollama
 
 
class OpenAIEmbeddingsLLM(BaseLLM[EmbeddingInput, EmbeddingOutput]):
    """A text-embedding generator LLM."""
 
    _client: OpenAIClientTypes
    _configuration: OpenAIConfiguration
 
    def __init__(self, client: OpenAIClientTypes, configuration: OpenAIConfiguration):
        self.client = client
        self.configuration = configuration
 
    async def _execute_llm(
        self, input: EmbeddingInput, **kwargs: Unpack[LLMInput]
    ) -> EmbeddingOutput | None:
        args = {
            "model": self.configuration.model,
            **(kwargs.get("model_parameters") or {}),
        }
        embedding_list = []
        for inp in input:
            embedding = ollama.embeddings(model="nomic-embed-text",prompt=inp)
            embedding_list.append(embedding["embedding"])
        return embedding_list
  1. 使用查詢功能(以下在terminal輸入) --method 可選local or global
python -m graphrag.query --root ./ragtest --method local "這個文章的主題是甚麼"

可能的Bug統整

自定義且文本大小很小

如果你的文本是自定義且文本大小很小記得調整下面的程式碼在setting.taml中否則會導致create_base_entity_graph階段出問題，以下的程式碼在設置texxtchunk的大小

chunks:
  size: 120 // change from 1200
  overlap: 10// change from 100
  group_by_columns: [id] # by default, we don't allow chunks to cross documents

使用global search時發生Keyerror

在"graphrag\graphrag\query\structured_search\global_search\search.py"中的235line那邊有格式的問題，在255行之前加一個
